Students API Documentation

The Students API allows you to manage student records via a simple RESTful interface. Below are the available endpoints:

Supported Formats

You can specify the format of the response by appending the format to the URL. The supported formats are json and xml.

For example:

  • /api/students.json - Returns data in JSON format.
  • /api/students.xml - Returns data in XML format.

GET /api/students

Retrieves a list of students. You can specify the response format (JSON or XML).

GET /api/students.{format}
Content-Type: application/json or application/xml
Response: 
[
  {
    "id": 1,
    "name": "John Doe",
    "phone": "123-456-7890"
  },
  ...
]

GET /api/students/{id}

Retrieves a single student by their ID. You can specify the response format (JSON or XML).

GET /api/students/{id}.{format}
Content-Type: application/json or application/xml
Response: 
{
  "id": 1,
  "name": "John Doe",
  "phone": "123-456-7890"
}

POST /api/students

Creates a new student record. The response can be in JSON or XML format.

POST /api/students
Content-Type: application/json or application/xml
Request Body: 
{
  "name": "Jane Doe",
  "phone": "987-654-3210"
}
Response: 
{
  "id": 2,
  "name": "Jane Doe",
  "phone": "987-654-3210"
}

PUT /api/students/{id}

Updates an existing student record by their ID. The response can be in JSON or XML format.

PUT /api/students/{id}.{format}
Content-Type: application/json or application/xml
Request Body: 
{
  "name": "John Doe",
  "phone": "123-456-7890"
}
Response: 
{
  "id": 1,
  "name": "John Doe",
  "phone": "123-456-7890"
}

DELETE /api/students/{id}

Deletes a student record by their ID. The response can be in JSON or XML format.

DELETE /api/students/{id}.{format}
Response: 
204 No Content
